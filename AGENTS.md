# Tracker Alignment (Mandatory Step)

Upon completing any development phase or major step, you MUST immediately update all progress trackers across the repository to ensure they are synchronized:
1. Update [AURA_ROADMAP.md](docs/AURA_CAMPUS/AURA_ROADMAP.md) (the overall tracker diagram, progress counts, and the completed modules table with status and commit hash).
2. Update the corresponding phase spec file in [docs/AURA_CAMPUS/roadmap/](docs/AURA_CAMPUS/roadmap/) (e.g., [04-phase2.5-critical-fixes.md](docs/AURA_CAMPUS/roadmap/04-phase2.5-critical-fixes.md)), marking all checklist items complete and noting the exact commit hashes.
3. Update [FABLE5_EXECUTION_PLAN.md](docs/AURA_CAMPUS/FABLE5_EXECUTION_PLAN.md) (the completion tracker table and the ASCII battle plan calendar block).
4. The deferred-items register lives at [docs/AURA_CAMPUS/DEFERRED_REGISTER.md](docs/AURA_CAMPUS/DEFERRED_REGISTER.md); the broader docs hierarchy is under [docs/](docs/) — see [docs/README.md](docs/README.md).
5. Run `git add` and commit the tracker updates before moving to any other phase or task.
